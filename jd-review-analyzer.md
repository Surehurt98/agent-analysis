# 京东商品好评分析器 (JD Review Analyzer)

## 🔇 静默执行协议 (Silent Execution Protocol)

> **CRITICAL INSTRUCTIONS FOR CLAUDE:**
> 
> 1. **Do NOT ask for confirmation to proceed.** Execute the entire workflow in one go.
> 2. **Do NOT ask "Should I continue?" or "Would you like me to..."** — Just do it.
> 3. **Generate the FULL output in one response.** Do not split or chunk the report.
> 4. **If data is missing or insufficient, use defaults and note it in the report.**
> 5. **The ONLY exception for pausing**: When login is required and NOT already logged in.

---

## 🎯 触发条件 (Trigger)

当用户输入包含以下模式时激活此 Skill：

- `分析京东链接：[URL]`
- `分析这个京东链接 [URL]`
- `分析京东商品 [URL]`

**URL 必须是京东商品详情页**（如 `https://item.jd.com/...`）

---

## ⚙️ 执行指令 (Execution Instructions)

### Step 1: 验证 URL

检查用户提供的 URL：

```
IF URL 为空:
    → 输出: "❌ 请提供京东商品链接"
    → END

IF URL 不是京东域名 (非 jd.com):
    → 输出: "❌ 仅支持京东商品链接，当前链接非京东域名"
    → END

IF URL 不是商品详情页 (不包含 item.jd.com 或 product):
    → 输出: "❌ 需要商品详情页链接，请勿使用搜索页或首页链接"
    → END
```

### Step 2: 打开商品页面

使用浏览器工具导航到商品 URL：

```
打开页面
IF 页面加载失败:
    → 重试 1 次
    IF 仍然失败:
        → 输出: "❌ 页面加载失败，请检查链接或稍后重试"
        → END
```

### Step 3: 提取商品基本信息

从页面中提取：
- 商品名称
- 当前价格
- 店铺名称
- 商品 ID (SKU)

```
IF 页面显示"商品已下架"或"商品不存在":
    → 输出: "❌ 该商品已下架或链接失效"
    → END
```

### Step 4: 导航到评论区

- 滚动页面或点击进入评论区
- 如果有"好评"筛选标签，点击选中

### Step 5: 检测登录状态 ⚠️ (唯一可中断点)

```
检查页面是否显示登录提示或登录弹窗

IF 需要登录 AND 当前未登录:
    → 输出: "⏸️ 需要登录才能查看评论。请在浏览器中完成登录，然后回复 **已登录** 或 **OK**"
    → WAIT for user response
    → 用户回复后继续

IF 已登录或无需登录:
    → 继续执行（不要输出任何内容）
```

### Step 6: 抓取评论

执行以下循环：

```python
collected_reviews = []
current_page = 1
max_pages = 5
target_count = 50

WHILE current_page <= max_pages AND len(collected_reviews) < target_count:
    reviews_on_page = 获取当前页所有评论
    
    FOR each review in reviews_on_page:
        IF review.rating == 5星 AND len(review.text) >= 10:
            collected_reviews.append(review)
            IF len(collected_reviews) >= target_count:
                BREAK
    
    IF 有下一页:
        点击下一页
        current_page += 1
    ELSE:
        BREAK
```

**注意：不要询问是否继续翻页，直接执行。**

### Step 7: 分析评论内容

对收集到的评论进行分析：

1. **高频关键词提取**：统计出现频率最高的产品相关词汇
2. **核心优点归纳**：基于关键词和评论内容，总结 TOP 5 核心优点
3. **代表性评论选取**：挑选 3-5 条最具代表性的原始评论

### Step 8: 生成报告

按照以下格式输出完整报告：

```markdown
# 📦 商品好评分析报告

## 📋 产品基本信息
- **商品名称**：[从页面提取]
- **当前价格**：¥[从页面提取]

- **商品ID**：[从页面提取]

## 🏆 核心优点 TOP 5
1. **[优点1标题]**：[具体说明，基于评论内容]
2. **[优点2标题]**：[具体说明]
3. **[优点3标题]**：[具体说明]

5. **[优点5标题]**：[具体说明]

## 🔑 用户高频关键词
`关键词1` `关键词2` `关键词3` `关键词4` `关键词5` `关键词6` ...

## 💬 代表性好评摘录
> "[评论原文1]" — 用户昵称
> 
> "[评论原文2]" — 用户昵称
> 
> "[评论原文3]" — 用户昵称

---
📊 **数据统计**：共分析 [N] 条5星好评（来自 [M] 页评论）
```

## ❌ 反面示例 (Anti-Patterns) — 严禁出现

### Anti-Pattern 1: 中途询问是否继续

```
❌ 错误示例：
"我已经收集了 30 条评论，要继续抓取更多吗？"
"评论比较多，需要我翻页继续吗？"
```

### Anti-Pattern 2: 询问输出格式

```
❌ 错误示例：
"你希望我用什么格式输出报告？Markdown 还是纯文本？"
"报告要包含哪些内容？"
```

### Anti-Pattern 3: 请求确认参数

```
❌ 错误示例：
"你要分析多少条评论？50 条够吗？"
"只看 5 星还是 4-5 星都要？"
```

### Anti-Pattern 4: 分段输出

```
❌ 错误示例：
"这是产品基本信息..."
"需要我继续分析评论吗？"
"这是优点总结..."
"要看原始评论摘录吗？"
```

### Anti-Pattern 5: 已登录时仍然询问

```
❌ 错误示例：
（页面已登录状态）
"检测到需要登录，请问您是否已经登录？"
```

## 🔧 默认配置 (Defaults)

| 参数            | 默认值   | 说明                     |
| --------------- | -------- | ------------------------ |
| `rating_filter` | 5星      | 仅抓取 5 星评论          |
| `min_length`    | 10字符   | 过滤短于 10 字的无效评论 |
| `max_reviews`   | 50       | 最多抓取 50 条有效评论   |
| `max_pages`     | 5        | 最多翻 5 页              |
| `output_format` | Markdown | 输出格式固定为 Markdown  |
| `language`      | 中文     | 报告语言                 |

---

## 🚫 边界情况处理 (Edge Cases)

| 情况           | 处理方式                     |
| -------------- | ---------------------------- |
| URL 为空       | 输出错误信息，不询问         |
| 非京东链接     | 输出错误信息，不询问         |
| 商品已下架     | 输出错误信息，不询问         |
| 无评论         | 输出"该商品暂无评论数据"     |
| 评论不足 50 条 | 继续分析，报告中注明数量     |
| 需要登录       | **唯一暂停点**，等待用户确认 |
| 翻页失败       | 使用已收集的评论继续分析     |
